# ゲーム画面UI要件書（クリア残り時間表示統合）

## 1. 目的
- ゲームクリアまでの残り時間をプレイ中に常時確認できるようにする。
- 既存のプレイヤーステータスUIと統合し、ゲーム画面UIを一体化する。

## 2. 対象範囲
- UI Toolkit（UXML / USS / C#）で構築するゲーム画面UI。
- 既存プレイヤーステータスUI（`GameScreenUiController`）への統合。

## 3. 機能要件
1. 残り時間表示
- ゲームクリアまでの残り時間をテキストで表示すること。
- 表示位置は画面中央上部とすること。
- 単位は時間仕様に合わせ、最低でも「分:秒（MM:SS）」形式で表示できること。

2. プレイヤーステータスUI統合
- 既存プレイヤーステータス表示（LV/HP/EXP/POW/DEF/SPD）を維持すること。
- 残り時間表示を同一ゲーム画面UI内で同時表示できること。
- 既存ステータス更新処理に影響を与えないこと。

3. UI Toolkit構成
- レイアウトはUXMLで定義すること。
- 見た目（位置・色・フォントサイズ・余白など）はUSSで定義すること。
- UI更新ロジックはC#コンポーネントで管理すること。

## 4. 非機能要件
1. 可読性
- 残り時間テキストはプレイ中に視認しやすい色・サイズ・コントラストを持つこと。

2. 保守性
- 既存UI構成に準拠した命名規則を用いること。
- ラベル参照は`name`ベースで取得し、ハードコードした階層依存を避けること。

3. パフォーマンス
- 毎フレーム更新が必要な場合でも、文字列更新は値変化時のみ反映する設計を推奨すること。

## 5. 画面要件
1. 画面構成
- 画面中央上部: 残り時間表示エリア（例: `09:59`）。
- 既存位置: プレイヤーステータスパネル（現在の配置を維持）。

2. レイアウト制約
- 解像度差異（16:9基準、他アスペクト比含む）で表示崩れしないこと。
- 残り時間表示は他UI要素と重なりにくいマージンを確保すること。

## 6. データ連携要件
- 残り時間表示UIは、ゲーム進行管理コンポーネントから現在の残り時間値を取得して表示すること。
- データ未取得時の初期表示値を定義すること（例: `--:--`）。

## 7. 受け入れ基準
1. プレイ中、画面中央上部に残り時間が表示される。
2. 残り時間が進行に応じて更新される。
3. 既存プレイヤーステータスUIが従来どおり表示・更新される。
4. UI Toolkit（UXML/USS/C#）のみで構築されている。
5. 複数解像度で重大なレイアウト崩れが発生しない。
