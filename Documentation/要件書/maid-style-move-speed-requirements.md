# メイドスタイル移動速度強化 要件書

## 背景と目的
既存のスタイル効果基盤では、スタイル変更時に `PlayerProgressionService` が対応する `IPlayerStyleEffect` を生成し、`PlayerState` に保持した倍率や状態を `PlayerController` が実効値へ反映する構成になっています。  
現状の `MaidStyleEffect` は追加効果を持たないため、メイドスタイルを選択しても固有のゲームプレイ変化が発生していません。  
本対応では、メイドスタイル選択中にプレイヤー移動速度を基準値の1.5倍へ補正し、スタイル変更時も常に現在スタイルの効果だけが反映される状態を保証します。

## スコープ内
- メイドスタイル選択時に、プレイヤーの実効移動速度へ1.5倍補正を適用する。
- メイドスタイル解除時に、移動速度補正を基準値へ戻す。
- 既存のスタイル切り替え処理の中で、他スタイルとの排他的適用を維持する。
- EditModeテストで、メイド選択時と解除時の倍率反映を検証可能にする。

## スコープ外
- メイドスタイルの移動速度以外の固有効果追加。
- `CharacterStatsData` 自体の基礎SPD値の変更。
- UI文言、演出、SE、VFXの追加変更。
- 敵キャラクターや他オブジェクトの移動速度への影響。

## 機能要件
1. メイド効果の適用
- `PlayerProgressionService.ChangeStyle(PlayerStyleType.Maid, context)` 実行時に、メイド用スタイル効果が `PlayerState.MoveSpeedMultiplier` を `1.5f` に設定する。
- メイド効果は `PlayerState` を通じて反映し、`PlayerController` の既存移動速度反映経路を再利用する。

2. 実効移動速度の反映
- プレイヤーの実効移動速度は、`CharacterStats.CurrentValues.Spd * PlayerState.MoveSpeedMultiplier` の結果として算出されること。
- メイド選択中は、同一 `CharacterStatsData` 基準値と比較して常に1.5倍の移動速度で移動すること。
- ステータス再反映が発生した場合でも、メイド効果適用中は1.5倍補正が維持されること。

3. スタイル切り替え時の解除
- メイドから別スタイルへ変更した際は、メイド由来の移動速度倍率が残存しないこと。
- 他スタイルからメイドへ変更した際は、旧スタイルの倍率や経験値補正が残存しないこと。
- 同じスタイルを繰り返し選択またはスタイル変更を連続で行っても、移動速度倍率が累積しないこと。

4. 既存スタイル効果との整合
- メイド効果は既存のスタイル効果基盤の責務内で実装し、UI層へ固有ロジックを追加しないこと。
- メイド効果は `Tick` ベースの継続処理を必要とせず、パラメータ適用のみで成立すること。

## 非機能要件
- 既存コード規約（暗黙private、`_camelCase`、公開APIへのドキュメントコメント）を順守する。
- 既存クラス責務を維持し、倍率管理は `PlayerState`、移動速度反映は `PlayerController`、スタイル固有値設定は `MaidStyleEffect` に限定する。
- 既存のアイドル移動速度補正との実装方針を揃え、特殊分岐を増やさない。
- 追加対応によりフレームごとの不要なアロケーションや継続処理を発生させない。

## 受け入れ条件
- メイドスタイル選択直後、プレイヤーの実効移動速度が基準SPDの1.5倍になる。
- メイドから巫女・ナース・海賊・カウガール・セレブ・アイドルのいずれかへ変更した直後、メイド由来の1.5倍補正が解除される。
- アイドルまたは他倍率効果スタイルからメイドへ変更した際、旧効果が残らずメイドの1.5倍だけが有効になる。
- スタイル変更を複数回繰り返しても、移動速度倍率が1.5倍を超えて累積しない。
- EditModeテストで、メイド選択時の `PlayerState.MoveSpeedMultiplier == 1.5f` と、他スタイルへの変更後に `1f` または変更先の期待値へ遷移することを確認できる。

## 未確定事項・確認事項
- 移動速度1.5倍は、ゲーム内表示や説明文に同期が必要か未確認です。本要件書ではゲームロジックのみを対象とします。
