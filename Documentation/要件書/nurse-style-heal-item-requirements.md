# ナース回復アイテム強化 要件書

## 背景と目的
ナーススタイルには現状固有効果が実装されておらず、`NurseStyleEffect` も空実装のままです。  
本対応では、ナース選択中のみ回復アイテム取得時の回復量を1.5倍にして、スタイルごとの差別化を明確にします。

## スコープ内
- ナーススタイル選択中、回復アイテム取得時の回復量に1.5倍補正を適用する。
- 他スタイルへ切り替えた時点で、回復アイテム回復量補正を解除する。
- 回復量補正の適用対象は `HealPickup` による回復アイテム取得のみとする。
- 既存のスタイル切替フロー（`PlayerController.ChangeStyle` -> `PlayerExperience.ChangeStyle`）に追従して効果を切り替える。

## スコープ外
- 巫女の定期回復や将来追加される別回復手段への倍率適用。
- 回復アイテムの基本回復量 `3` 自体のバランス変更。
- ナース固有のUI演出、SE、VFX追加。
- スタイル選択UI文言の改修。

## 機能要件
1. ナース効果の適用
- `PlayerStyleType.Nurse` への変更時、回復アイテム回復量倍率を `1.5` に設定する。
- 効果適用後も `PlayerState.CurrentStyleType` はナースとして保持される。
- 同一スタイルを再適用しても倍率が重複加算されない。

2. 回復アイテム取得時の回復量計算
- `HealPickup` はプレイヤー回復時、基礎回復量 `3` に現在の回復アイテム倍率を乗算した値を使用する。
- 回復量は整数として `Mathf.RoundToInt` で算出する。
- 基礎回復量 `3` に倍率 `1.5` を適用した結果、実回復量は `5` とする。
- 最終的なHP反映は既存 `HealthComponent.Heal(int amount)` の上限処理に従い、最大HPを超えない。

3. スタイル切替時の解除
- ナースから他スタイルへ変更した時点で、回復アイテム倍率を初期値 `1.0` に戻す。
- ナース切替前にフィールド上へ生成済みの回復アイテムも、取得時点のプレイヤー側倍率を参照して回復量を決定する。
- ナース以外のスタイルでは、回復アイテム回復量は常に基礎値 `3` 相当へ戻る。

4. 適用対象の限定
- 本倍率は `HealPickup.CollectPickup` 経由の回復にのみ適用する。
- `MikoStyleEffect` の定期回復、デバッグ回復、将来の別API呼び出しには自動適用しない。

## 非機能要件
- 既存コード規約（暗黙private、`_camelCase`、公開APIへのドキュメントコメント）を順守する。
- スタイル効果の責務は既存の `PlayerProgressionService` / `IPlayerStyleEffect` 系へ寄せ、`HealPickup` にスタイル判定を直接書き込まない。
- スタイル未設定や参照未解決時は、既存挙動を壊さず倍率 `1.0` として扱える実装にする。
- 毎フレーム更新で不要なアロケーションを発生させない。

## 受け入れ条件
- ナース選択中に回復アイテムを取得すると、基礎回復量 `3` に対して実際の回復量が `5` になる。
- ナース選択中でも、回復後HPは最大HPを超えない。
- ナースから別スタイルへ変更後に回復アイテムを取得すると、回復量が基礎値 `3` に戻る。
- ナース選択前に生成された回復アイテムを、ナース選択後に取得した場合も回復量 `5` が適用される。
- 巫女の定期回復はナース効果の影響を受けず、既存値のまま動作する。

## 未確定事項・確認事項
- 回復アイテム倍率は `PlayerState` に新規プロパティとして保持する案を第一候補とする。
- `HealPickup` から倍率参照する経路は、`PlayerController` 参照経由と `PlayerStyleEffectContext` 拡張のどちらを採るか設計時に確定する。
- 端数処理は `Mathf.RoundToInt(3 * 1.5f) = 5` を採用する前提。切り捨て運用へ変える場合は受け入れ条件も更新が必要。
