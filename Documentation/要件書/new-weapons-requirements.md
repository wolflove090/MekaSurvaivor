# 新規武器追加 要件書

## 背景と目的
- 現在の戦闘システムには `BulletWeapon`、`ThrowingWeapon`、`DamageFieldWeapon` の3系統のみが実装されている。
- ゲームプレイの選択肢を増やすため、挙動の異なる新規武器として「ドローン」「バウンドボール」「火炎瓶」を追加する。
- 既存の武器基盤（`WeaponBase`、`WeaponService`、`IWeaponEffectExecutor`、各種 `*Controller`）を活かしつつ、追加武器を既存のアップグレード導線と両立できる形で設計する。

## スコープ内
- 新規武器3種の要件定義。
- 既存の武器生成フローへ新規武器を組み込むための拡張方針整理。
- 武器アップグレード導線に新規武器を提示するために必要なUI・列挙値・管理テーブル拡張の要件整理。
- 既存の `ProjectileController` / `DamageFieldController` で再利用できない挙動を補う新規 `Application` / `Infrastructure` 層責務の整理。

## スコープ外
- 武器の最終的な数値バランス調整（詳細なダメージ量、クールダウン、射程、見た目サイズの微調整）。
- 新規武器の演出アセット制作（モデル、VFX、SE、アニメーション）の完成。
- 既存3武器の仕様変更。
- ステージ地形仕様そのものの刷新。

## 機能要件

### 共通
- 新規武器は既存の `WeaponBase.Tick` ベースのクールダウン管理、または同等の周期発動管理に統合される。
- 新規武器はプレイヤーの攻撃力（`Pow`）をダメージ計算へ反映できる。
- 新規武器は既存武器と同様に、取得済みならレベルアップ、未取得なら武器チェーンへ追加される。
- 新規武器の追加に伴い、`WeaponUpgradeUiController.UpgradeCardType` と `WeaponService` の武器ビルダー管理を拡張できる構造にする。
- 武器強化UIは表示枠を3枚のまま維持し、その時点で候補となる武器からランダムに3件を表示する。

### ドローン
- ドローンはプレイヤー周囲を継続的に周回する追従オブジェクトとして出現する。
- ドローン本体は一定距離を保ちながらプレイヤーに追従し、一定周期で敵を探索する。
- 発射時はドローン位置を起点に最寄りの有効敵へ向けて弾を射出する。
- 攻撃対象が存在しない場合、ドローンは周回のみ継続し、無駄撃ちはしない。
- ドローンの発射弾は既存通常弾を流用するか、ドローン専用弾を用意するかを実装計画で確定する。
- レベルアップ時は発射間隔が短くなる。

### バウンドボール
- バウンドボールは発射時、プレイヤー位置から右下方向を初期進行方向として射出される。
- 射出物は敵または破壊可能オブジェクトに衝突した際、即時消滅せず反射方向ベースで進行方向を変えて継続する。
- バウンド回数は最大3回までとし、3回目のヒット後は消滅またはプール返却される。
- 1回のヒットごとに、直前の接触面または接触方向をもとに反射方向を再計算する。
- 右下方向はワールド座標基準とし、初期方向は `Vector3.right` と下方向成分の合成ベクトルで決定する。
- 各ヒット時にダメージを発生させ、既存の `IDamageable` へ適用できる。

### 火炎瓶
- 火炎瓶はプレイヤー位置からプレイヤーの向きに応じて投擲され、着地点に到達した時点でダメージエリアを生成する。
- 射出中の火炎瓶本体はダメージを発生させない。
- 投擲軌道は放物線を描く挙動として扱い、見た目と着地点が一致する。
- 着地判定は、射出物の `y` 座標が地面オブジェクトから取得した基準 `y` に到達したタイミングで行う。
- 着地後に生成される炎エリアは一定時間その場に残り、接触中の敵へ継続ダメージを与える。
- 炎エリアは既存 `DamageFieldController` の追従しない派生仕様、または等価な新規コンポーネントで実現する。
- 炎エリアは敵だけでなく破壊可能オブジェクトにも継続ダメージを与える。
- 炎エリアは持続時間経過後に消滅またはプール返却される。
- レベルアップ時は少なくとも持続時間延長、範囲拡大、ダメージ間隔短縮のいずれかを強化対象にできる設計にする。

## 非機能要件
- 既存の `Application` / `Infrastructure` 分離方針を崩さず、新規武器ロジックは可能な限り `Application` 層で発動条件を組み立てる。
- 武器追加後も `PlayerController.InitializeWeaponSystems` の責務が過度に肥大化しないよう、武器組み立てロジックは `WeaponService` または専用ビルダーへ集約する。
- 追加した投射物・追従オブジェクトは `ObjectPool` を利用し、生成破棄コストを抑える。
- 武器候補のランダム表示は、未取得武器と取得済み武器の強化候補を破綻なく混在できる選定ロジックにする。
- EditMode テストで、武器追加の分岐、バウンド回数制御、継続ダメージ条件など主要ロジックを検証できる状態にする。
- 既存武器の挙動回帰を避けるため、新規武器追加による既存列挙値・既存UI動作の破壊を防ぐ。

## 受け入れ条件
- 新規武器3種がそれぞれ仕様どおりの発動挙動を持ち、既存ゲームループ内で発動する。
- ドローンはプレイヤーに追従しながら周回し、敵がいる時のみ一定周期で射撃する。
- ドローンのレベルアップ後は発射間隔が短縮される。
- バウンドボールは初期方向がワールド基準の右下であり、最大3回のヒットまで反射しながら複数回ダメージを与えられる。
- 火炎瓶は飛翔中にダメージを与えず、プレイヤーの向きに応じて飛翔し、着地後にのみ継続ダメージエリアを生成する。
- 火炎瓶のダメージエリアは一定時間で消滅し、破壊可能オブジェクトにも継続ダメージを与える。
- 武器取得・強化導線から新規武器を選択可能で、既存武器の取得・強化も継続して機能する。
- 追加または更新したテストで、新規武器の主要条件分岐が検証される。

## 未確定事項・確認事項
- バウンドボールの反射方向は、衝突法線ベースで計算する。
- 武器強化UIのランダム表示で、候補不足時の補完ルール（重複許可有無、取得済み優先度）を実装計画で確定する必要がある。
