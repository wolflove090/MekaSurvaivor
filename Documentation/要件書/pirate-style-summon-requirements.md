# 海賊スタイル召喚能力 要件書

## 背景と目的
現在の `PirateStyleEffect` は未実装で、海賊スタイルを選択しても固有能力は発生しない。  
本対応では、海賊スタイル選択中のみ一定間隔で戦闘員を召喚し、敵の攻撃対象をプレイヤーから戦闘員へ一時的に切り替えることで、生存補助として機能するスタイル能力を追加する。

## スコープ内
- 海賊スタイル有効中、10秒ごとに戦闘員を2体召喚する。
- 戦闘員は敵方向へ移動する。
- 敵はプレイヤーより戦闘員を優先して攻撃対象にする。
- 戦闘員は敵へ攻撃しない。
- 海賊スタイル解除時に、新規召喚を停止し、存在中の戦闘員を破棄する。

## スコープ外
- 戦闘員の攻撃手段追加。
- 戦闘員専用のSE、VFX、アニメーション追加。
- 海賊スタイル以外から戦闘員を召喚する仕組みの共通化。
- 戦闘員ステータスのゲーム内調整UI追加。

## 機能要件
1. 海賊スタイル効果の発動条件
- `PlayerStyleType.Pirate` 選択中のみ召喚能力を有効化する。
- 海賊スタイルへ変更後、ゲーム内時間10秒経過ごとに召喚を1回実行する。
- 1回の召喚で戦闘員を2体生成する。
- 海賊スタイルから別スタイルへ変更した時点で、次回以降の召喚タイマー進行と新規召喚を停止する。
- 海賊スタイルから別スタイルへ変更した時点で、存在中の戦闘員を全て破棄する。

2. 戦闘員の生成
- 戦闘員はプレイヤー前方の扇状位置に生成する。
- 戦闘員はプレイヤー陣営オブジェクトとして扱い、敵と区別できる識別を持つ。
- 戦闘員のHPは30とする。
- 戦闘員は再利用可能な生成方式を優先し、短周期生成で不要なGCを増やさない。
- 戦闘員の生成失敗時は、ゲーム進行を止めず警告ログで検出可能にする。
- 次回の2体召喚時、前回までに存在していた戦闘員は召喚処理の中で破棄する。

3. 戦闘員の移動
- 戦闘員は生成後、敵方向へ自律移動する。
- 戦闘員は最寄りの敵を逐次再探索し、その敵へ向かって移動する。
- 有効な敵が存在しない場合は、キャラクター前方へ直進する。
- 移動は既存ゲーム空間のXZ平面ルールに従う。

4. 敵ターゲット優先制御
- 敵は有効な戦闘員が存在する間、プレイヤーより戦闘員を優先して追跡・攻撃する。
- 複数の戦闘員が存在する場合、どの戦闘員を選ぶかは既存の最短距離ベースを優先候補とする。
- 優先対象の戦闘員が無効化または消滅した場合、敵は次の有効な戦闘員を参照し、存在しない場合はプレイヤー追跡へ戻る。
- 既に出現済みの敵にも優先制御を反映し、新規スポーンした敵と挙動差が出ないようにする。

5. 戦闘員の非攻撃制約
- 戦闘員は通常攻撃、接触ダメージ、投射物生成など、敵へダメージを与える手段を持たない。
- 戦闘員は敵の注意を引きつけることと移動のみを責務とする。

## 非機能要件
- 既存コード規約（暗黙private、`_camelCase`、公開APIへのドキュメントコメント）を順守する。
- スタイル効果ロジックに召喚物の実体処理を過度に埋め込まず、責務を分離する。
- `Time.timeScale` が0の間は召喚タイマーを進めない。
- 敵ターゲット切替は毎フレーム不要なアロケーションを発生させない。
- 参照切れした戦闘員を敵が追跡し続けないよう、null安全に実装する。

## 受け入れ条件
- 海賊スタイル選択後、ゲーム内時間10秒ごとに戦闘員が2体ずつ生成される。
- 海賊スタイル以外では戦闘員が生成されない。
- 戦闘員は最寄りの敵へ向かって移動し、敵がいない場合はキャラクター前方へ移動する。
- 戦闘員が存在する間、敵はプレイヤーより戦闘員を優先して追跡する。
- 戦闘員が全て無効化または消滅した後、敵はプレイヤー追跡へ戻る。
- 戦闘員から敵へダメージが発生しない。
- スタイルを海賊から他スタイルへ変更すると、新規召喚が停止し、存在中の戦闘員が破棄される。
- 戦闘員のHPは30である。
- 次回召喚時に前回の戦闘員が破棄されるが、破棄タイミングによって一時的に4体存在しても進行不良にならない。
