# 武器レベル段階調整 要件書

## 背景と目的
- 現在の武器レベルアップ挙動は、`BulletWeapon`、`ThrowingWeapon`、`BoundBallWeapon`、`FlameBottleWeapon`、`DroneWeapon` が「割合短縮ベース」の強化を採用しており、指定したレベルごとの固定値に一致しない。
- `DamageFieldWeapon` は段階的な拡大を持つが、今回の指定値に合わせて各レベルのサイズ定義を明確化する必要がある。
- 武器ごとにレベル1からレベル5までの強化内容を固定値で管理し、ゲーム内の強化結果を仕様どおりに揃える。

## スコープ内
- 既存6武器のレベル1-5の性能テーブル再定義。
- 既存 `WeaponBase` 派生クラスの `LevelUp()` および初期状態の見直し。
- 固定クールダウン、サイズ、バウンド回数、同時展開数、投射物数に関する実装方針の整理。
- 既存実装と新仕様の不一致箇所を明示し、計画書作成に必要な確認事項を整理する。

## スコープ外
- 武器ダメージ量、弾速、ノックバック、VFX、SEの調整。
- 新規武器種の追加。
- 武器レベルアップUIの見た目変更。
- レベル6以上の拡張仕様定義。

## 機能要件

### 共通
- 各武器は、取得直後をレベル1として扱い、初期性能が下記レベル1の値に一致する。
- 各武器のレベルアップは、現在値に割合で補正するのではなく、レベルに対応した固定値へ直接更新する。
- 既存の最大レベル5の制約は維持し、レベル5到達後の追加強化では性能を変化させない。
- `WeaponUpgradeUiController` および `WeaponService` の既存強化導線から見た「未取得なら取得、取得済みなら1レベル上昇」の基本挙動は維持する。

### シューター（`BulletWeapon`）
- レベルごとの発射間隔は以下に固定する。
- レベル1: 1.5秒
- レベル2: 1.0秒
- レベル3: 0.8秒
- レベル4: 0.4秒
- レベル5: 0.2秒
- `BulletWeapon` の取得直後に、内部クールダウン基準値が1.5秒として扱われる。

### スロー（`ThrowingWeapon` として扱う）
- 本要件では、入力仕様の「スロー」は既存コード上の `ThrowingWeapon` を指すものとして扱う。
- レベルごとの発射間隔は以下に固定する。
- レベル1: 2.0秒
- レベル2: 1.5秒
- レベル3: 1.0秒
- レベル4: 0.8秒
- レベル5: 0.4秒

### ダメージフィールド（`DamageFieldWeapon`）
- レベルごとのサイズは以下に固定する。
- レベル1: 3.0
- レベル2: 3.5
- レベル3: 4.0
- レベル4: 4.5
- レベル5: 5.0
- サイズは `DamageFieldSpawnRequest.AreaScale` に渡る値として反映される。
- 強化時に生成間隔は変更しない。

### ドローン（`DroneWeapon`）
- レベルごとのドローン攻撃間隔は以下に固定する。
- レベル1: 2.0秒
- レベル2: 1.5秒
- レベル3: 1.0秒
- レベル4: 0.8秒
- レベル5: 0.4秒
- レベル1-4では同時展開数を1機とする。
- レベル5では同時展開数を2機に増やす。
- レベル5で2機展開する際は、2機とも同一半径を周回し、円周上で180度反対の位置に配置して重ならないようにする。
- レベル5の2機化に伴い、既存の `WeaponEffectExecutor` にある単一アクティブドローン前提（`_activeDroneController` とプール数1）は見直し対象とする。

### バウンドボール（`BoundBallWeapon`）
- 発射方向は、現在の「右下固定」ではなく、プレイヤー位置から真下方向へ変更する。
- レベルごとの性能は以下に固定する。
- レベル1: 発射間隔2.0秒、最大1回バウンド
- レベル2: 発射間隔1.5秒
- レベル3: 発射間隔1.0秒、最大2回バウンド
- レベル4: 発射間隔0.8秒
- レベル5: 発射間隔0.4秒、最大3回バウンド
- レベル2およびレベル4で明示されていないバウンド回数は、直前レベルの設定値を維持する。
- 入力仕様の「レベル4: 0.8s + 発射間隔を短く」は、0.8秒指定と重複しているため、要件書では「レベル4の発射間隔は0.8秒」として整理し、追加の短縮挙動は未確定事項に残す。

### 火炎瓶（`FlameBottleWeapon`）
- レベルごとの性能は以下に固定する。
- レベル1: 投擲間隔1.5秒、投射物数1
- レベル2: 投擲間隔1.0秒、投射物数1
- レベル3: 投射物数2
- レベル4: 投射物数3
- レベル5: 投射物数4
- レベル3-5で投擲間隔の再指定がないため、レベル2以降の投擲間隔は1.0秒を維持する。
- 複数投射物化に伴い、1回の発動で複数回 `FireFlameBottle()` を実行できる構成、または等価な複数投射要求の表現が必要である。
- 複数投射物は、プレイヤー前方を中心とした扇状拡散で投擲する。

## 非機能要件
- 既存の `Application` / `Infrastructure` 分離方針を維持し、武器性能テーブルは `WeaponBase` 派生クラスまたはそれに準ずるアプリケーション層の責務として整理する。
- レベルごとの固定値は、`LevelUp()` 内で累積誤差が出ない更新方法にする。
- クールダウン短縮時は既存どおり現在クールダウンを新しい持続時間に合わせて補正し、即時多重発射や長すぎる待機を防ぐ。
- ドローン2機化および火炎瓶複数投射により、既存のプール数不足や単一インスタンス前提で破綻しないようにする。
- EditMode テストで、主要なレベル境界値と特別挙動（ドローン2機化、バウンド回数、火炎瓶投射数）を検証できる状態にする。

## 受け入れ条件
- `BulletWeapon` のレベル1-5で発射間隔が 1.5 / 1.0 / 0.8 / 0.4 / 0.2 秒になる。
- `ThrowingWeapon` のレベル1-5で発射間隔が 2.0 / 1.5 / 1.0 / 0.8 / 0.4 秒になる。
- `DamageFieldWeapon` のレベル1-5でサイズが 3.0 / 3.5 / 4.0 / 4.5 / 5.0 になる。
- `DroneWeapon` のレベル1-5で攻撃間隔が 2.0 / 1.5 / 1.0 / 0.8 / 0.4 秒になり、レベル5で同時に2機展開される。
- レベル5の2機ドローンは、同一半径上の正反対の位置を維持して周回する。
- `BoundBallWeapon` がプレイヤー真下へ発射され、レベル1・3・5でそれぞれ最大1回・2回・3回バウンドする。
- `FlameBottleWeapon` の投擲間隔がレベル1で1.5秒、レベル2以降で1.0秒となり、レベル3-5で投射物数が2・3・4に増える。
- `FlameBottleWeapon` の複数投射は、プレイヤー前方中心の扇状拡散で発射される。
- 既存の武器取得・強化フローからこれらの性能が適用され、最大レベル5を超えても異常動作しない。
